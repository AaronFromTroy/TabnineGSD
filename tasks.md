# Task Breakdown

This file defines how tasks are expanded and validated.

## Task Structure
Each task must include:
- Description
- Inputs
- Outputs
- Dependencies
- Risks
- Acceptance criteria

## Example Template
### Task: Implement Authentication Service
- Inputs: user credentials
- Outputs: JWT token
- Dependencies: database, hashing library
- Risks: security, token expiration
- Acceptance: passes tests, meets security requirements
